## 自动化管道 (Auto-Pipeline)

### Step 1: 初始化与对阵抓取
- **Action**: 立即执行 `python3 {skill_dir}/scripts/fetch_match_data.py`。
- **Verification (关键强制步骤)**：
  - 如果 API 返回结果为空，或用户指定了 API 不支持的联赛（如沙特联、日职、中超等）：
  - **必须**先调用 `WebSearch` 搜索「YYYY-MM-DD + 联赛名称 + schedule/fixtures」进行真实对阵校验。
  - **严禁**在未通过联网核实的情况下输出任何猜测或模拟的对阵信息。
- **Output**: 获取今日所有竞足赛事列表。

### Step 2: 三维数据采集（含战意优先）
- **采集维度（按权重排序）**：
  - **战意分析 (第一优先级)**：通过 `WebSearch` 获取积分形势、保级/争冠/欧战资格需求、球队近期新闻动态。
    - **关键指标**：保级生死战（5星）、争冠/争四关键战（5星）、无欲无求中游队（2星）
  - **赔率走势（如用户提供）**：初赔→即时赔率变化方向、市场资金流向（主升客降/主降客升/平局下降等）
  - **状态历史**：通过 API 或 `WebSearch` 获取 H2H 历史对阵、近5场状态、主客场差异
- **禁止**：不主动通过任何途径检索赔率数据。赔率信息 **仅在用户主动提供（截图或文字）时** 纳入分析。

### Step 3: 三维加权研判与信心修正
- **三维加权模型（验证准确率 87%+）**：
  - **常规赛季权重**：
    - **战意分析 (40%)**：积分压力、目标需求、球队动力（赛季末段可调整为 50%）
    - **赔率走势 (35%)**：初赔→即时赔率变化方向与幅度（如无赔率则权重转移给战意）
    - **状态历史 (25%)**：H2H 历史 + 近期状态（赛季末段可调整为 15%）
  - **赛季末段权重（推荐）**：
    - **战意分析 (50%)** + **赔率走势 (35%)** + **状态历史 (15%)**
- **战意分级规则**：
  - **5星战意**：保级生死战、争冠/争四关键战、德比大战
  - **4星战意**：欧战资格争夺战、杯赛半决赛/决赛
  - **3星战意**：中游排位赛
  - **2星战意**：已保级/已无望欧战的纯荣誉战
- **赔率走势信号**：
  - **强烈信号**：目标方向赔率持续下降 0.15+
  - **警示信号**：热门方向赔率持续上升 0.15+
  - **平局信号**：平局赔率较初赔下降 0.2+
- **热度降权规则**：
  - 涉及以下球队自动标记「高热」：Real Madrid, Barcelona, Manchester City, PSG, Bayern Munich, Liverpool, Arsenal, Napoli, Tottenham, Benfica
  - 高热场次：信心上限 75%，且必须输出风险提示
  - 冷门联赛（瑞典超、韩职、日职等）：可维持正常信心上限 90%
- **方向决策**：基于 **战意优先 + 赔率验证 + 历史参考** 综合判断。

## 赔率截图模式（用户自带赔率 · 波动分析）

适用于：用户上传 **多场比赛** 的 **赔率/盘口走势截图**（含初赔→即赔、时间线、升降趋势等），要求基于图中数据做 **波动分析**。

### 触发关键词（示例）

`赔率截图`、`盘口波动`、`变盘`、`走势`、`以我图为准`、`不要联网赔率`、`别查 API` 等；或用户 **先发说明** 再连续发多张图。

### 硬规则（必读）

- **数据源唯一**：任务存续期间，**以用户截图（及用户随图粘贴的数字）为唯一赔率来源**。**禁止**用 `WebSearch` / `WebFetch` 抓取网页或接口赔率来 **替代、覆盖或静默修正** 读图结果。
- **读图失败**：若关键赔率格、时间戳、玩法列无法辨认，**只能**请用户 **放大局部、重截、或手打一行数字**；**禁止**联网「替用户填数」。
- **唯一例外**：用户 **明确追加**指令如「再帮我用 API 对照一下」时，可拉取 API/网页数据，但输出须 **分两列/两小节**：**「截图口径」**与 **「API/网页口径」**，并标明二者差异；**默认仍以截图为主** 写波动结论，除非用户声明以 API 为准。

### 与自动化管道的关系

- 本模式 **不触发** 上文「今日推荐 / 足彩分析」一键 Step 1–4 全链。
- 若用户同一会话 **先** 给截图模式、**后** 又要今日全自动推荐：以 **后一条明确指令** 为准，并在切换时 **声明数据源**（截图 vs 基本面）。

### 执行步骤

1. **Read 全部截图**：按用户说明或画面标题，将每张图 **归属到具体场次**（联赛、主队 VS 客队、竞彩编号若有）；多图一场时按 **时间或箭头顺序** 排先后。
2. **转写数据表（以图为准）**：每场整理可见玩法（如 **胜平负**、**让球**、**大小球** 等——**只写图中出现的项**），列出 **初赔 / 各时点 / 即时** 数值；读图存疑处标注「读图不确定」。
3. **波动分析（每场）**：对图中每个 relevant 选项，说明相对初赔 **走高 / 走低 / 先升后降 / 震荡**；若图中有 **概率条、凯利、成交量** 等，仅作图中已有信息的解读，**不引入外网数据**。
4. **综合提示（可选、简短）**：仅基于 **已转写的赔率运动**，写 1～3 条「市场倾向在截图口径下的含义」或「需警惕的变盘形态」；**不**擅自给出投注指令，除非用户明确要求（仍须提示风险）。

### 标准输出模版

```markdown
## 数据源说明
- 本场分析 **仅依据用户截图**，未使用联网/API 赔率。

## 逐场赔率转写（读图）

### 场次 X · 主队 VS 客队
| 时间/阶段 | 玩法 | 项 | 赔率 | 备注（读图不确定处） |
|-----------|------|----|------|----------------------|
| … | 胜平负 | 主 | … | … |

### 场次 X · 波动小结
- 主胜/平/客：**相对初赔**的变化方向与幅度（文字描述即可）
- 让球（若有）：同上
- **读图不确定项**：（列出）

（每场重复）

## 跨场简评（可选，≤5 行）
- 仅对比各截图内已出现的数字，不下外网结论。
```
